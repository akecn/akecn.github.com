排版字体比例

整个排版体系基于两个在variables.less文件中定义的变量：@baseFontSize 和 @baseLineHeight。第一个变量定义了所有字体大小的基础值，而第二个是行高的基础值。

我们用这些变量配合数学计算,创造所有我们需要的外边距（margins），内补白（paddings），行高（line-heights）等等。

表单

灵活的HTML和CSS

表单中最棒的就是所有的输入框和表单控件不论你怎么去摆放，看起来都非常棒。我们提供了一些模式来实现效果，而不需要去添加没必要的HTML元素。

更复杂的布局结构通过简洁灵活的样式名来渲染样式以及事件绑定。

布局类型

Bootstrap包含4种表单布局类型：

  • 垂直对齐 （默认）
  • 搜索类型
  • 内联
  • 水平对齐

不同类型的表单布局需要对结构做一些修改，但是表单控件本身和它的表现是不变的。

表单控件的状态显示和其他

Bootstrap的表单部分包括所有你知道的基础表单控件，像输入框（input）、多行输入框（textarea）和下拉菜单（select）。它还增加了一些自定义的组件，如输入框的前置/后置文本以及复选框列表。

像错误（error）、警告（warning）以及成功（success）等是每个表单控件都有的状态。当然还包含了不可用状态（disabled）的显示。

4种表单类型

Bootstrap提供了简单的结构和设计来显示常见的web表单。

类型名称 样式名 描述
垂直对齐 （默认） .form-vertical （不是必须的） 分层的，做对齐的表单控件
内联 .form-inline 向左对齐并在同一行内显示完成的表单
搜索类型 .form-search 为搜索框附加典型的圆角效果
水平对齐 .form-horizontal 每行向左浮动，标签文本向右对齐

表单样式使用示例只使用表单元素而不增加任何其他的标签元素

      </div>
    </div>
  </fieldset>
</form>

表单控件状态

Bootstrap的一个特色是浏览器支持的聚焦和不可用disabled状态的样式展示。我们去除了Webkit浏览器默认的outline效果并增加了盒阴影box-shadow来替代它原本的聚焦效果:focus。

前置 & 后置的输入框

表单组—带有前置或后置文本—为你的输入框提供了一种简便的方式来更好的传达用意。最佳的例子包括Twitter用 @ 符号来标识用户名，还有用 $ 标识资产。

复选框和单选框

升级到1.4版本以后，Bootstrap需要用额外的标签包裹复选款和单选框来将它们组合起来。而现在，之需要简单的用<label class="checkbox">标签来包裹<input type="checkbox">就可以了。

同行显示的复选框和单选框也是被支持的。只需要给.checkbox或.radio标签增加样式名.inline就行了。

Inline forms and append/prepend

在同一行内显示前置或后置文本输入框，需要确保.add-on的元素和input元素放在同一行，并且它们之间不能有空格。

为行为创建按钮

作为约定，按钮标签（button）应当只用于动作，而超链接则用于对象的描述。举个例子，“下载”应该是一个按钮（button）而“最近的活动”则应该是一个链接。

凡是使用.btn样式名的元素都会有按钮的样式。但是应当只使用<a>和<button>元素来作为按钮显示。

跨浏览器兼容

IE9浏览器尚不支持背景渐变（background gradients），所以我们移除了对它的支持。另外IE9下对于button元素disabled状态下的难看的灰色阴影是我们无法修正的。

IE9 doesn't crop background gradients on rounded corners, so we remove it. Related, IE9 jankifies disabled button elements, rendering text gray with a nasty text-shadow that we cannot fix.

构造sprite图

为了避免让每一个图标都占用一个额外的请求，我们将它们合并到一个sprite图中—在一个文件中排列多张图片并利用CSS的背景定位background-position来显示对应的图片。我们在Twitter.com中也是这么使用的，对我们来说这中方式相当有效。

所有图标的样式都有.icon-前缀来标识，就像我们对其他组件样式名的定义一样。这可以避免发生样式名冲突的情况。

Glyphicons已经授权我们在开源的工具集中使用他们的作品，只要我们在文档中给出来源链接并说明版权。希望你们在项目中也能这么做。

如何使用

Bootstrap使用<i>标签作为所有图标的显示容器，但并没有限定样式名—除了那个约定的前缀。使用的时候，只需要将与下面类似的代码放到任何你需要显示的位置即可：

<i class="icon-search"></i>

还可以使用反色（白色）的图标样式，只需要额外增加一个样式名即可：

<i class="icon-search icon-white"></i>

这里总共有120个图标供你选择。只需要添加<i>标签并正确设置你想要的图标样式即可。你可以在sprites.less文件或者本文档中找到完整的样式列表。

注意！ When using beside strings of text, as in buttons or nav links, be sure to leave a space after the <i> tag for proper spacing.

范例

图标非常棒，但是可以在哪里使用呢？这里提供部分建议：

  • 为你的侧边导航栏添加视觉效果
  • 用于创建纯图标的导航栏
  • 传达按钮操作的含义
  • 指示链接的跳转的目标内容

理论上讲，任何你能放置<i>标签的地方，你都可以放置一个图标。
